# /fix-review コマンド

指定されたPRのレビュー指摘を取得し、修正対応を行います。

## 入力

- `$ARGUMENTS`: PR番号 または PR URL

## 手順

### 1. PR情報の取得

```bash
gh pr view $ARGUMENTS --json number,title,headRefName,baseRefName
```

PR のブランチにチェックアウトしてください。

### 2. レビューコメントの取得

以下のコマンドでレビュー情報を取得してください。

```bash
# レビュー一覧（承認/変更要求/コメントの状態）
gh api repos/{owner}/{repo}/pulls/{number}/reviews

# インラインコメント（ファイル・行番号付きの指摘）
gh api repos/{owner}/{repo}/pulls/{number}/comments
```

### 3. 指摘内容の解析

取得したコメントを以下の基準で分類してください:

- **must**: 必ず修正が必要（`[must/` を含むコメント、または REQUEST_CHANGES のレビュー）
- **imo**: 修正を推奨（`[imo/` を含むコメント）
- **nits**: 軽微な提案（`[nits/` を含むコメント）
- **その他**: 上記に該当しない通常のコメント

**対応不要なコメントを除外**:
- 既に「修正しました」等の返信があるコメントスレッド
- bot による自動コメント

### 4. 修正方針の提案

各指摘について以下をまとめ、ユーザーに提示してください:

```
## レビュー指摘一覧

### must（X件）
1. [ファイル:行番号] 指摘内容
   → 修正方針: ...

### imo（X件）
1. [ファイル:行番号] 指摘内容
   → 修正方針: ...

### nits（X件）
1. [ファイル:行番号] 指摘内容
   → 修正方針: ...

上記の方針で修正してよろしいですか？
```

**ユーザーの承認を得てから修正に進んでください。**

### 5. コード修正

承認後、各指摘に対応するコードを修正してください。

- CLAUDE.md のコンベンションに従う
- 修正は指摘内容に対応する最小限の変更に留める
- 型チェック `npx tsc --noEmit` を実行して問題がないことを確認

### 6. コミットと push

```bash
git add <修正ファイル>
git commit -m "Fix review comments"
git push
```

- コミットメッセージは修正内容を端的に表す
- 修正の粒度が大きい場合は複数コミットに分ける

### 7. レビューコメントへの返信

修正をコミットした後、`git rev-parse HEAD` でコミットハッシュを取得してください。

返信を送信する前に、各コメントへの返信内容をユーザーに提示して確認を得てください:

```
## 返信内容の確認

1. [comment_id: XXX] ファイル:行番号
   > 元の指摘内容
   返信: 修正しました。 {commit_sha}
   {修正内容の説明}

2. ...

上記の内容で返信してよろしいですか？
```

**ユーザーの承認を得てから返信を送信してください。**

承認後、修正した各指摘に対して、コミットハッシュと修正内容の説明を含めて返信してください。

```bash
COMMIT_SHA=$(git rev-parse HEAD)

gh api repos/{owner}/{repo}/pulls/{number}/comments \
  -X POST \
  -f body="修正しました。 ${COMMIT_SHA}

{修正内容の説明}" \
  -F in_reply_to=<comment_id>
```

#### 返信フォーマット

```
修正しました。 <commit_sha>

<どのように修正したかの簡潔な説明>
```

例:
```
修正しました。 a1b2c3d

nullチェックを追加し、undefinedの場合は早期リターンするようにしました。
```

- 各指摘ごとに、何をどう変えたかを1〜2文で説明する
- 対応しなかった指摘がある場合はその理由を説明する

### 8. 結果サマリーの表示

```
## 修正完了

PR: #{number} {title}

| 分類 | 指摘数 | 修正済 | スキップ |
|------|--------|--------|----------|
| must | X件    | X件    | X件      |
| imo  | X件    | X件    | X件      |
| nits | X件    | X件    | X件      |

修正をpushし、レビューコメントに返信しました。
```

## 注意事項

- must の指摘は原則すべて修正してください
- imo / nits はユーザーの判断でスキップ可能
- 指摘の意図が不明な場合はユーザーに確認してください
- 修正によって既存テストが壊れないことを確認してください
